Esta aplicación es una SPA (Single Page Application) que interactúa con una API para gestionar operaciones CRUD de recetas (Crear, Leer, Actualizar y Borrar) en una base de datos relacional alojada en PostgreSQL. Además, es una aplicación de recetas que incluye funcionalidades avanzadas como búsquedas, filtrados, ordenamientos y creación de registros. Este conjunto de tecnologías garantiza una experiencia de usuario fluida y una gestión eficaz de los datos, ofreciendo así una aplicación robusta y de alto rendimiento

Esta aplicación proporciona una plataforma integral para la gestión de Historias Clínicas,incluye búsquedas, filtrados y ordenamientos eficaces. Además, ofrece acceso a tres tipos de usuarios distintos: Paciente, Médico y Administrador. Gestión de Usuarios y Autenticación segura mediante Auth0. Integración fluida con la pasarela de pago Mercado Pago para transacciones eficientes. Notificaciones automatizadas por correo electrónico para mantener a los usuarios informados. Sistema de Administración de Usuarios para una gestión eficaz de perfiles y permisos. Dashboard intuitivo que proporciona una visión completa de la información relevante. El proyecto se desarrolló siguiendo la metodología ágil SCRUM, garantizando así una planificación efectiva y una entrega precisa. Tecnologías Utilizadas: Lenguaje: JavaScript Base de Datos: PostgreSQL con Sequelize como ORM Backend: Node.js con Express.js Frontend: React y Redux Estilos: CSS, Bootstrap
